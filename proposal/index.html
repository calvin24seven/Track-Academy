04 / Diagnosis

The current site is holding Track Academy back.

A short list of specific things we'd change first. Not a critique â€” making the opportunity obvious.

Registration is a wall, not a doorway

The Upshot form has 50+ fields, demographic dropdowns for ethnicity and faith before a parent can even enter their child's name. Parents abandon it. Staff answer the phone instead. It's the number one friction point.

The Contact page still has Lorem ipsum

trackacademy.co.uk/contact-us literally contains placeholder text. It's the kind of detail that erodes trust in seconds â€” and exactly the kind of detail a funder would notice.

Impact is only visible in a PDF

The 2024 Annual Report has all the numbers: 577 sessions, 331 young people, 66% improved school performance, 98% recommendation rate. None of it appears on the website in a way that updates or routes funders to it.

The homepage speaks to no one specifically

Parents, schools, and funders all see the same hero and the same four cards. There's no "I'm a parent, I'm from a school, I'm a funder" routing. Every audience has to work to find their own relevant info.

Events and fixtures are manually edited

Staff edit WordPress pages when events change. Fixtures are in a table updated by hand. There's no filter, no archive, no database. It's maintenance work dressed up as content.

The website has group photos but no individual bios

We have the assets: named staff, coaches, mentors, and trustees are all documented. These just need to be connected to their photos on the site. Funders researching Track Academy should see who runs it â€” not guess from a group shot.

07 / Registration

Replace the 50-field form with a step-by-step wizard.

The current Upshot embed asks a Brent parent about their 8-year-old's GCSE results before they've entered their child's name. We fix it â€” without touching Upshot's data.

Today

  • 50+ fields on a single page
  • Ethnicity, faith, gender before the child's name
  • Education fields shown regardless of child's age
  • No save-and-continue
  • No sibling autofill
  • No mobile-optimised layout
  • Validation only on submit
  • Confusing for Universal Credit applicants
  • Lost on refresh or accidental back
  • No confirmation email with next steps

Rebuild

  • 7-step wizard â€” one thing at a time
  • Child's name and DOB first, auto assigns age group
  • "Register another child" pre-fills parent details
  • Mobile-first, large tap targets, one column
  • Inline validation â€” friendly, on blur
  • Universal Credit flow auto-applies free place
  • Server-side autosave â€” never lose progress
  • Confirmation email: session time, what to bring, Spond link
How it syncs with Upshot: The new registration saves to our own Supabase database first. A sync job then maps the fields to Upshot's expected format and pushes them via Upshot's API. If the sync fails, it retries and flags it â€” the parent sees a clean confirmation either way, and Track Academy staff are alerted to fix it. Upshot gets the data. The parent gets a real form.
